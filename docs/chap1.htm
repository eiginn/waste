Chapter 1 Introduction

1.1                    Major Functions

PadLockSL (PadLock Secured Link) is a tool that is designed to allow users to communicate and transfer information in a secure way. PadLockSL uses a peer to peer authorization which requires very little administration, and no central server or central administration is required.

PadLockSL provides a generic virtual secured private network, and implement following services to use on the network:

  Instant Messaging - allows users to communicate with other users on a private     

    PadLockSL network in the same way as using AIM/ICQ/IRC/etc. This feature is primarily

    accessed through the main PadLockSL Window.

  Group Chat - allows two or more users to chat on a PadLockSL network in the same

    way as using AIM/ICQ/IRC/etc. This feature is primarily accessed through the

    PadLockSL main Window.

  Distributed presence - allows users to see what other users are currently on a private 

    PadLockSL network. This feature is primarily accessed through the main PadLockSL 

    Window, and facilitates ease in Instant Messaging.

  File browsing - allows users to browse a virtual directory structure for each user on the

    network. Each user can specify a list of directories to make available to other users on the

    network. This feature is primarily accessed through the PadLockSL Browser Window.

  File transfer - allows users to transfer files to or from other users. Files can be found via the

    file browsing and file searching features, or files can be uploaded to other users manually.

    This feature is accessed through many interfaces, and can be managed with the

    PadLockSL File Transfer window.

  Key distribution - allows users on the PadLockSL network to exchange public keys so that

    they can directly connect to each other (which helps the network optimize itself)

1.2                    Special Characteristics

PadLockSL utilizes hardware AES algorithm and random number generator provided in VIA C5P processor. The special characteristics PadLockSL has are outlined as below:

1.2.1    Support running on C5P system and non-C5P system

1.2.2    Automatically detect whether C5P ACE is available or not

   If C5P ACE is available, use hardware AES in C5P ACE; otherwise, use software  

    implemented AES when performing AES encryption/decryption

1.2.3    Automatically detect whether C5P RNG is available or not

   If C5P RNG is available, use it as entropy source in random number generation

    routine; otherwise, use the random number generation device provided by linux.

1.2.4    AES key length support which pertain to software and hardware implemented AES

   AES key length is configurable

   AES key length actually used in a connection is the maximum of AES key lengths configured

    In two sides of a connection

   Default key length is 128 bits

   Support 192 bits key length

   Support 256 bits key length

1.2.5    AES cipher mode support which pertain to software and hardware implemented AES

   Support CBC mode
